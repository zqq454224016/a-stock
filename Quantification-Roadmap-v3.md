# AI 量化系统 v3 稳定化与扩展需求文档

> 日期：2026-07-10
> 基线：参考 `Quantification.md v2.34`。MVP / v2 首版主线已归档；工程优化与模块拆分见 `Quantification-Optimization.md`。
> V3-01/V3-02 已完成首版。本文只承接 **能力扩展** 的唯一执行顺序，不替代总需求文档与优化文档。

## 1. v3 目标

把 v2 的首版能力推进到可长期运行、可扩展股票池、可审计外部智能、可持续评估的稳定化系统。

核心原则：

- 不恢复自动实盘，实盘相关仍保持只读、模拟、人工确认和审计优先。
- 先补运行可靠性和数据血缘，再扩大股票池和模型能力。
- 所有新增智能输出必须可解释、可回放、可降级。
- 全市场扩展必须先过数据质量、市场范围、流动性和风险过滤。

## 4. V3-01 运行时任务日志（已完成首版）

### 目标

记录每次任务的开始、结束、耗时、状态、异常和产物路径，使监控不再只依赖产物更新时间代理任务状态。

### 输入

- `main.py`
- `scheduler`
- `tasks`
- `monitoring`

### 输出

- `assets/data/task_runs/*.json`
- `assets/data/task_runs/index.json`
- `reports/monitoring/` 中的最近任务状态和失败原因

### 验收标准

- 所有 CLI 任务可写入 task_run 记录。
- 成功、失败、跳过三类状态都有结构化输出。
- 失败任务能关联异常摘要、运行耗时、输入命令和相关产物。
- `monitor` 报告展示最近任务状态、失败原因和最近一次成功时间。
- 单元测试覆盖成功任务、失败任务和记录裁剪。

### 已落地实现

- 新增 `task_runs` 运行日志存储，每次 CLI 命令记录 `run_id`、命令参数、开始/结束时间、耗时、状态、异常摘要和关键产物状态。
- `assets/data/task_runs/index.json` 保留最近 100 次运行摘要，便于监控页快速读取。
- `monitor` 快照新增最近任务、失败次数、上一任务状态和任务失败告警；`reports/monitoring/index.html` 新增“最近任务”表。
- 当前覆盖 CLI 入口；长驻 `scheduler` 内部子任务独立 run_id 作为 V3-02/V3-06 的延伸优化。

## 4.1 V3-02 数据注册表与血缘版本（已完成首版）

### 目标

把“任务运行记录”继续延伸到“数据产物注册表”：每个关键 JSON/HTML 产物都能说明来源任务、生成时间、输入依赖、字段摘要、数据质量、降级原因和版本快照。

### 输入

- `storage/json_store.py`
- `monitoring/task_runs.py`
- `monitoring/snapshot.py`
- 各 `tasks/*_job.py` 输出产物

### 输出

- `assets/data/data_registry/*.json`
- `assets/data/data_registry/index.json`
- `reports/monitoring/` 中的数据注册表与动态血缘

### 验收标准

- 每个关键 JSON 产物有来源任务、更新时间、生成命令、字段摘要和记录数量。
- 数据降级、缓存命中、补录来源可追踪。
- 监控血缘从静态模块清单升级为“任务运行记录 + 数据注册表”联合推导。
- 单元测试覆盖注册表写入、缺失产物、降级原因和血缘摘要。

### 已落地实现

- 新增 `registry` 模块与 `registry` 命令，生成 `assets/data/data_registry/*.json` 和 `assets/data/data_registry/index.json`。
- 注册 20 个关键产物，记录存在性、文件大小、更新时间、记录数、字段摘要、降级状态、限制、依赖和最近生成任务。
- `monitor` 自动先刷新注册表，再生成监控快照；监控报告新增“注册表血缘”。
- 当前血缘由静态依赖 + 最近 CLI 任务日志推导；缓存命中、补录来源和字段级质量仍需后续任务写入更细元数据。

## 4.2 当前唯一下一步：V3-03 全市场股票池首版

### 目标

从固定 watchlist 扩展到可配置、可过滤、可解释的股票池体系，同时保持科创板默认参考模式，避免直接扩大重型拉取和策略主链路风险。

### 输入

- `watchlist`
- `market_scope`
- `quality`
- `factors`
- `data_registry`

### 输出

- `assets/data/universe/*.json`
- `assets/data/universe/index.json`
- `reports/console/` 中的股票池筛选入口

### 验收标准

- 支持 watchlist、行业池、风险排除池、参考池。
- 默认不重型拉取科创板，仅保留参考标的。
- 推荐、selector 和 attribution 可选择股票池而不是固定自选股。
- 注册表记录股票池来源、过滤原因、样本数量和更新时间。

## 5. v3 开放风险

| 风险 | 影响 | 处理方式 |
|------|------|----------|
| 外部数据源和网络不稳定 | 影响实时性和全市场覆盖 | 保留本地缓存优先、降级记录和数据注册表 |
| 当前推荐范围仍以自选股为主 | 推荐不代表全市场机会 | V3-03 前不扩大推荐承诺 |
| 真实 LLM、外部通知、券商只读网关需要密钥 | 存在权限和安全风险 | 密钥只走环境变量或本地私有配置，不写入报告和 Git |
| 横截面因子检验需要更大样本 | 当前小样本结论可能偏差 | 先扩大股票池，再做行业/市值中性化 |
