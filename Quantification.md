## 一、系统总体架构

### 1.1 核心数据流

```
爬虫层（行情 / 舆情 / 研报 / 产业链）
        ↓
数据清洗治理层（P0）
        ↓
统一数据仓库（MySQL / TSDB / Redis）
        ↓
因子计算引擎（技术 / 情绪 / 机构 / 产业）
        ↓
策略层（选股 / 择时 / 风控）
        ↓
回测引擎
        ↓
实盘交易系统
        ↓
AI Agent 分析层
        ↓
Web 可视化平台
```

### 1.2 与现有工程映射（`quant_system/`）

| 层级 | 当前模块 | 状态 |
|------|----------|------|
| 爬虫层 | `data_source/`（东财 / 新浪降级、分钟线） | 部分完成 |
| 清洗治理 | `pipeline/cleaner`, `validator`, `normalizer` | 基础完成 |
| 存储 | `storage/json_store`（Redis / MySQL 预留） | JSON 已用 |
| 盘中分析 | `pipeline/intraday_analyzer`, `tasks/intraday_job` | 已完成 |
| 日级分析 | `pipeline/stock_analyzer`, `tasks/stock_job` | 已完成 |
| 调度 | `scheduler/cron_runner` | 已完成 |
| 因子 / 回测 / 实盘 / Agent | — | 待建设 |

## 二、阶段拆分

### Phase 0：数据基建（P0）

> **目标**：让所有数据「可用、可信、可计算」——必须优先完成。

#### 2.1 数据清洗与标准化

**功能模块**

- K 线异常数据过滤（涨跌停异常、停牌、0 成交量）
- 去重（分钟级 / 日级）
- 缺失数据补齐
- 股票代码统一（6 位标准）
- 交易日历统一（A 股）

**标准 K 线结构**

```json
{
  "ts": "",
  "code": "",
  "open": 0,
  "high": 0,
  "low": 0,
  "close": 0,
  "volume": 0,
  "amount": 0,
  "adj_factor": 1
}
```

#### 2.2 复权系统

- 前复权
- 后复权
- 分红送股修复
- `adj_factor` 版本管理

#### 2.3 数据校验与质量监控（建议补充）

- 跨源一致性校验（东财 vs 新浪）
- 延迟 / 缺失告警
- 数据版本与采集时间戳

#### 2.4 统一数据仓库

- MySQL：元数据、财报、研报
- TSDB：分钟 / Tick 时序
- Redis：盘中实时缓存（`live:stock:{code}`）

#### 2.5 基础数据补齐

- PE / PB / ROE
- 财报数据
- 龙虎榜
- 解禁 / 分红 / 业绩预告
- 指数数据（沪深 300 / 中证 500）
- 利率数据

### Phase 1：多源数据扩展（P1）

#### 3.1 舆情系统（雪球为核心）

**数据结构**

```json
{
  "code": "",
  "title": "",
  "content": "",
  "likes": 0,
  "comments": 0,
  "author": "",
  "time": "",
  "platform": ""
}
```

**情绪分类**：看多 / 看空 / 中性

**输出指标**

- 多空比
- 热度指数
- 情绪加速度

#### 3.2 机构研报系统

**采集字段**

- 目标价
- 评级变化
- 分析师信息
- 覆盖机构数

**输出因子**

- 目标价溢价率
- 一致预期价
- 评级动量

#### 3.3 产业链题材系统

**重点赛道**

- AI
- 半导体
- 电子特气
- 稀缺金属

**输出**

- 赛道 → 事件 → 个股映射
