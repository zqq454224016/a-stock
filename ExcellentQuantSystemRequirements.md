## 一、系统目标

一个优秀的量化系统不只是“会算指标”或“能跑回测”，而是要形成完整闭环：

```text
数据采集
  → 数据治理
  → 因子计算
  → 策略研究
  → 回测验证
  → 预测输出
  → 风控过滤
  → 模拟 / 实盘执行
  → 绩效归因
  → 复盘迭代
```

系统最终应支持：

- 稳定采集并治理多源金融数据。
- 快速开发、验证和比较量化因子。
- 支持可复现、接近真实交易约束的回测。
- 基于历史验证输出股票走势预测，而不是无依据判断。
- 支持模拟交易和受控实盘交易。
- 通过 AI Agent 提供解释、诊断、复盘和投研辅助。
- 通过 Web 看板统一展示行情、因子、策略、预测、风险和绩效。

## 二、优秀量化系统应具备的核心能力

| 能力 | 说明 | 是否核心 |
|------|------|----------|
| 数据可信 | 数据完整、字段统一、可追溯、可校验 | 是 |
| 因子可复用 | 因子有版本、参数、公式、输入输出规范 | 是 |
| 回测可信 | 支持 T+1、手续费、滑点、涨跌停、停牌、复权 | 是 |
| 策略可复现 | 固定数据、参数、版本后结果一致 | 是 |
| 预测可验证 | 预测必须关联历史样本、胜率、收益风险比 | 是 |
| 风控前置 | 回测、模拟、实盘共用风控规则 | 是 |
| 执行可审计 | 下单、撤单、拒单、风控拦截均可回放 | 是 |
| 结果可解释 | 输出信号来源、因子贡献、风险因素、失效条件 | 是 |
| 监控可告警 | 数据失败、任务失败、异常收益、风险超限可告警 | 是 |
| 扩展可维护 | 数据源、因子、策略、存储、交易接口可插拔 | 是 |

## 三、总体架构

```text
data_source/      行情、财报、舆情、研报、产业链、宏观数据
pipeline/         清洗、标准化、复权、质量评分、补爬
storage/          JSON / MySQL / TSDB / Redis / 对象存储
factors/          技术、基本面、情绪、机构、产业、资金因子
strategy/         选股、择时、调仓、止盈止损、组合构建
backtest/         撮合、成交约束、绩效分析、归因分析
prediction/       走势预测、概率输出、置信度、风险标记
risk/             仓位、回撤、集中度、流动性、黑名单、熔断
portfolio/        持仓、现金、净值、敞口、再平衡
trading/          模拟交易、实盘接口、订单、成交、账户同步
agent/            数据巡检、策略诊断、选股解释、复盘报告
web/              行情看板、因子排名、预测看板、回测报告、实盘监控
scheduler/        定时任务、依赖编排、失败重试、运行日志
monitoring/       指标监控、告警、审计、权限、配置管理
```

## 四、功能需求

### 4.1 数据系统

**目标**：覆盖量化研究和交易所需的核心数据，并支持多源降级。

**应支持的数据**

- 行情数据：日 K、分钟线、Tick、盘口、成交明细。
- 指数数据：宽基指数、行业指数、主题指数。
- 基本面数据：财报、估值、盈利能力、成长性、资产负债。
- 公司行为：分红、送转、配股、停复牌、解禁、退市、ST。
- 资金数据：成交额、换手率、主力资金、北向资金、融资融券。
- 舆情数据：新闻、公告、雪球 / 股吧讨论、热度、情绪。
- 机构数据：研报、评级、目标价、覆盖机构、盈利预测。
- 产业链数据：题材、事件、上下游关系、价格变化。
- 宏观数据：利率、汇率、商品价格、流动性指标。

**关键要求**

- 所有数据必须记录 `source`、`fetched_at`、`raw_version`。
- 采集失败必须可重试、可告警、可降级。
- 多源数据不可静默覆盖，必须保留来源和差异记录。
- 数据应区分原始层、清洗层、特征层和结果层。

### 4.2 数据治理与质量系统

**目标**：保证进入因子和回测的数据可信。

**功能**

- 股票代码标准化。
- 交易日历统一。
- 缺失值检查与补齐。
- 重复数据去重。
- 异常值检测。
- 前复权 / 后复权。
- 停牌、涨跌停、零成交识别。
- 跨源一致性校验。
- 数据质量评分。
- 低质量数据隔离。

**质量评分示例**

```json
{
  "code": "000001",
  "trade_date": "2026-07-03",
  "missing_rate": 0,
  "duplicate_count": 0,
  "cross_source_diff": 0.001,
  "quality_score": 98,
  "status": "ok"
}
```

**质量门禁**

- `quality_score >= 90`：允许进入因子、回测和预测。
- `70 <= quality_score < 90`：允许展示，回测需显式允许。
- `quality_score < 70`：隔离，不参与策略计算。

### 4.3 因子系统

**目标**：把原始数据转化为可复用、可比较、可验证的量化信号。

**因子类型**

- 技术因子：MA、MACD、RSI、KDJ、ATR、波动率、动量、突破。
- 量价因子：成交额、换手率、量价背离、资金流强度。
- 基本面因子：PE、PB、ROE、毛利率、净利率、营收增速。
- 情绪因子：多空比、热度、评论增速、情绪分歧。
- 机构因子：目标价空间、评级变化、覆盖机构数、盈利预测变化。
- 产业因子：事件强度、涨价强度、供需紧张度、题材热度。
- 风险因子：波动率、回撤、流动性、集中度、尾部风险。

**因子规范**

- 因子必须有名称、公式、参数、版本、频率和适用范围。
- 因子不得使用未来数据。
- 因子输出必须关联数据版本。
- 因子变更必须生成新版本。
- 缺失、停牌、上市不足 N 日等情况必须有明确处理规则。

### 4.4 策略研究系统

**目标**：支持从信号到策略的研发、比较、迭代。

**策略类型**

- 技术趋势策略。
- 均值回归策略。
- 多因子选股策略。
- 事件驱动策略。
- 情绪反转 / 情绪动量策略。
- 行业轮动策略。
- 大盘择时策略。
- 组合再平衡策略。

**策略配置**

- 股票池。
- 选股条件。
- 买入条件。
- 卖出条件。
- 调仓频率。
- 最大持仓数。
- 单票仓位。
- 止盈止损。
- 风控过滤。

**策略输出**

- 候选股票池。
- 交易信号。
- 仓位建议。
- 触发原因。
- 风险标记。
- 策略版本。

### 4.5 回测系统

**目标**：尽可能接近真实交易环境，验证策略是否有效。

**必须支持**

- T+1 交易规则。
- 手续费、印花税、滑点。
- 涨跌停不可成交。
- 停牌不可交易。
- 前复权价格回测。
- 成交量 / 成交额流动性约束。
- 股票池动态变化。
- 退市、ST、新股过滤。
- 多策略对比。
- 回测结果可复现。

**绩效指标**

- 总收益。
- 年化收益。
- 最大回撤。
- 夏普比率。
- 胜率。
- 盈亏比。
- 换手率。
- 持仓周期。
- 超额收益。
- 信息比率。
- 收益回撤比。

**回测输出**

- 资金曲线。
- 持仓明细。
- 交易明细。
- 每日净值。
- 每笔交易原因。
- 风控拦截记录。
- 策略参数快照。
- 数据 / 因子 / 策略版本。

### 4.6 预测系统

**目标**：基于已验证策略和因子，输出可验证、可解释的股票走势预测。

**预测类型**

- `1d`：短线方向预测。
- `5d`：周级趋势预测，适合作为 MVP。
- `20d`：月级趋势预测。
- 盘中预测：基于分钟线和实时资金变化。

**预测输出**

```json
{
  "code": "000001",
  "trade_date": "2026-07-03",
  "horizon": "5d",
  "direction": "up",
  "probability": 0.61,
  "confidence": "medium",
  "expected_return": 0.028,
  "max_expected_drawdown": 0.045,
  "evidence": {
    "backtest_win_rate": 0.57,
    "profit_loss_ratio": 1.35,
    "sample_count": 420
  },
  "risk_flags": ["low_liquidity", "sector_high_volatility"]
}
```

**预测要求**

- 预测必须关联策略版本、因子版本和数据版本。
- 预测必须输出概率和置信度，不允许输出确定性结论。
- 样本不足时只能输出低置信度或拒绝预测。
- 预测必须支持事后复盘，统计命中率、收益和失效原因。

### 4.7 风控系统

**目标**：风控必须前置，并贯穿回测、模拟和实盘。

**风控规则**

- 单票最大仓位。
- 行业 / 题材集中度限制。
- 最大回撤熔断。
- 单日亏损限制。
- 流动性过滤。
- 波动率过滤。
- ST / 退市 / 黑名单过滤。
- 重大事件窗口降仓。
- 高频交易频率限制。
- 账户级风险敞口控制。

**风控输出**

- 是否放行。
- 拦截原因。
- 建议仓位。
- 风险等级。
- 风控规则版本。

### 4.8 模拟交易与实盘交易系统

**目标**：把策略从研究结果转化为可执行交易流程。

**模拟交易**

- 虚拟账户。
- 模拟订单。
- 模拟成交。
- 模拟持仓。
- 模拟收益曲线。
- 与回测结果对比。

**实盘交易**

- 券商接口适配。
- 下单、撤单、查询。
- 持仓和资金同步。
- 订单状态追踪。
- 成交回报处理。
- 交易前风控。
- 交易后复盘。

**实盘约束**

- 默认不允许自动下单。
- 开启自动交易必须显式授权。
- 所有订单必须可审计、可回放。
- 风控拒单不可绕过。

### 4.9 组合管理系统

**目标**：管理多股票、多策略、多账户的组合表现。

**功能**

- 持仓管理。
- 现金管理。
- 仓位计算。
- 组合净值。
- 行业暴露。
- 风格暴露。
- 策略贡献。
- 调仓建议。
- 再平衡。
- 组合归因。

**输出**

- 当前持仓。
- 目标持仓。
- 调仓差异。
- 风险敞口。
- 策略贡献。
- 收益归因。

### 4.10 AI Agent 系统

**目标**：提供自动化分析、诊断、解释和复盘能力。

**Agent 类型**

- 数据巡检 Agent：发现缺失、异常、延迟，并触发补爬。
- 因子分析 Agent：检查因子分布、漂移、失效。
- 策略诊断 Agent：分析策略收益来源、回撤原因、参数敏感性。
- 选股解释 Agent：解释为什么某只股票进入股票池。
- 舆情解读 Agent：总结市场热点、情绪分歧和事件影响。
- 复盘 Agent：对预测、交易和风控结果做事后复盘。

**权限边界**

- Agent 默认只能分析和建议。
- Agent 不得默认直接下单。
- Agent 输出必须附带证据链。
- Agent 行为必须记录输入、输出、引用数据和执行结果。

### 4.11 Web 可视化系统

**目标**：让用户能直观看到系统状态、股票机会、风险和绩效。

**页面模块**

- 市场总览。
- 自选股看板。
- 个股详情页。
- 因子排名。
- 策略信号。
- 走势预测。
- 回测报告。
- 模拟交易。
- 实盘监控。
- 风控面板。
- 数据质量面板。
- Agent 分析报告。

**个股详情页应展示**

- 基础行情。
- K 线和均线。
- 技术因子。
- 基本面摘要。
- 舆情热度。
- 机构观点。
- 预测方向和概率。
- 风险标记。
- 历史信号表现。
- 当前策略建议。

### 4.12 监控、审计与运维系统

**目标**：保障系统稳定运行，并支持问题定位。

**监控**

- 数据采集成功率。
- 数据延迟。
- 任务运行耗时。
- 回测 / 预测任务失败率。
- 实盘订单失败率。
- 风控拦截次数。
- 系统资源使用情况。

**告警**

- 数据源失效。
- 行情延迟。
- 质量评分过低。
- 任务连续失败。
- 回撤超限。
- 订单异常。
- 账户资金异常。

**审计**

- 数据版本审计。
- 策略参数审计。
- 回测结果审计。
- 预测输出审计。
- 订单和成交审计。
- Agent 操作审计。

## 五、阶段规划与能力交付

| 阶段 | 名称 | 核心交付 | 能力边界 |
|------|------|----------|----------|
| Phase 0 | 数据基建 | 数据采集、清洗、复权、质量评分、交易日历 | 能判断数据是否可信，不做预测 |
| Phase 1 | 因子 MVP | 技术因子、因子版本、因子排名 | 能输出趋势倾向，不作为交易预测 |
| Phase 2 | 回测 MVP | 单策略回测、交易成本、绩效指标、交易明细 | 能验证策略历史有效性 |
| Phase 3 | 预测 MVP | `5d` 方向、概率、置信度、风险标记 | 开始具备可验证走势预测 |
| Phase 4 | 多因子与多策略 | 多因子组合、策略对比、参数优化 | 预测能力更稳健 |
| Phase 5 | 模拟交易 | 虚拟账户、模拟订单、模拟持仓、复盘 | 验证真实运行稳定性 |
| Phase 6 | 实盘辅助 | 券商接口、交易前风控、订单审计 | 可辅助实盘，不默认自动交易 |
| Phase 7 | AI Agent | 策略诊断、选股解释、舆情解读、复盘报告 | 提升解释和迭代效率 |
| Phase 8 | 产品化平台 | Web 看板、权限、监控、告警、审计 | 形成可持续使用的系统 |

## 六、走势预测能力达成路径

### 6.1 能力分级

| 等级 | 阶段 | 能力 | 是否可用于交易 |
|------|------|------|----------------|
| L0 | 数据基建 | 只判断数据质量 | 否 |
| L1 | 因子 MVP | 输出因子倾向和趋势信号 | 否 |
| L2 | 回测 MVP | 验证信号历史表现 | 可用于研究 |
| L3 | 预测 MVP | 输出概率、置信度和风险 | 可用于人工辅助 |
| L4 | 模拟交易 | 验证实时运行和执行偏差 | 可用于模拟盘 |
| L5 | 实盘辅助 | 接入真实账户和风控 | 可用于小资金实盘辅助 |
| L6 | AI 解释 | 输出证据链、失效条件和复盘 | 可用于投研决策 |

### 6.2 最小可用预测定义

系统至少完成以下内容，才可以称为具备股票走势预测能力：

- 数据通过质量门禁。
- 因子无未来函数。
- 策略经过回测。
- 回测包含交易成本、T+1、停牌、涨跌停。
- 预测输出方向、概率、置信度、样本数量和风险标记。
- 预测结果支持事后复盘。

### 6.3 MVP 预测目标

近期最小目标建议为：

- 预测周期：`5d`。
- 股票范围：自选股或 100 只核心股票。
- 策略基础：技术趋势策略。
- 输出字段：方向、概率、置信度、预期收益、预期回撤、风险标记。
- 验证方式：滚动回测 + 预测复盘。

---

## 七、非功能需求

### 7.1 可复现性

- 固定数据、参数、因子版本、策略版本后，结果必须一致。
- 每次回测、预测、交易都必须记录版本快照。

### 7.2 可扩展性

- 新数据源可插拔。
- 新因子可注册。
- 新策略可配置。
- 新交易接口可适配。
- 存储层可从 JSON 平滑迁移到数据库。

### 7.3 稳定性

- 任务失败可重试。
- 数据异常可隔离。
- 外部数据源不可用时可降级。
- 实盘模块出现异常时必须停止自动交易。

### 7.4 性能

- 日线级回测应支持批量股票。
- 因子计算应支持增量更新。
- Web 看板应优先读取预计算结果。
- 高频或 Tick 能力应独立规划，不影响 MVP。

### 7.5 安全与权限

- 实盘交易权限必须单独配置。
- 敏感配置不得写入代码。
- 操作日志不可随意删除。
- Agent 不得绕过权限和风控。

---

## 八、验收标准

### 8.1 数据验收

- 数据字段统一。
- 最近 N 个交易日无严重缺失。
- 质量评分可生成。
- 异常数据可定位来源。

### 8.2 因子验收

- 因子输出稳定。
- 因子无未来函数。
- 因子版本可追溯。
- 因子缺失处理明确。

### 8.3 回测验收

- 支持交易成本。
- 支持 T+1。
- 支持停牌和涨跌停。
- 输出交易明细和资金曲线。
- 同一输入多次运行结果一致。

### 8.4 预测验收

- 输出方向、概率、置信度和风险。
- 预测关联回测证据。
- 样本不足时拒绝高置信度输出。
- 可统计预测命中率和收益表现。

### 8.5 实盘验收

- 所有订单经过风控。
- 所有订单状态可追踪。
- 所有成交可回放。
- 交易异常可告警。

---

## 九、风险与边界

- 系统输出不构成投资建议。
- 回测收益不代表未来收益。
- 预测是概率判断，不是确定性结论。
- 数据延迟、字段变更、采集失败可能导致信号失真。
- 实盘交易必须由用户授权，并保留人工干预能力。
- Agent 默认只做分析和建议，不直接执行交易。
- 第三方数据采集需遵守对应平台服务条款。
