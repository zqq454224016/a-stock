> 日期：2026-07-10  
> 基线：参考 `Quantification.md v2.33`，当前系统已具备数据、因子、回测、预测、selector、impact、每日涨跌归因、decision、review、replay、portfolio、因子历史代理评估、多周期推荐、模块化契约、Agent Provider 框架、统一控制台、监控告警与数据血缘、运行时任务日志、数据注册表、v3 稳定化路线和静态报告 MVP。  
> 本文只整理未完成主线任务、可优化项和下一步唯一执行顺序，不替代总需求文档。

## 1. v2 目标

把当前系统从“可运行的研究闭环”推进到“可复盘、可校准、可接实盘辅助、可解释推荐”的量化系统。

核心原则：

- 不直接自动交易，所有实盘相关动作先进入只读、模拟、人工确认和审计链路。
- 所有推荐必须带证据、风险、失效条件和后验复盘入口。
- 十日前视角推演必须禁止未来函数，真实收益只用于事后复盘。
- 科创板继续保持参考模式，减少重型拉取和策略主链路占用。

## 3. 可优化项补充

| 模块 | 可优化项 | 原因 | 优先级 |
|------|----------|------|--------|
| review | 当前样本大量 pending 时无法直接评估命中率 | 最新信号缺少未来 K 线，需用 replay 兜底校准 | P0 |
| replay | 根因分析已生成，但尚未全面反馈到 prediction/selector/decision | 需要形成“错因 → 阈值/权重调整”的闭环 | P0 |
| selector | 阈值原先固定，无法随近期命中率变化 | 已新增自动校准，后续需按市场环境分层校准 | P0 |
| decision | 空仓/半仓/满仓建议已有雏形，但缺少价格级买卖触发 | 后续需延伸为“到达多少买/卖/止损/失效” | P1 |
| impact | 事件提取仍偏摘要，缺少公告全文与产业价格真实序列 | 对利通电子利润增长、昊华科技材料涨价等事件判断仍需更强证据 | P1 |
| portfolio | 账户级风险已有，行业/风格/流动性暴露不足 | 实盘前必须知道风险集中在哪些维度 | P1 |
| factor | 已有历史代理评估，但尚非标准横截面检验，也未行业中性化 | 小股票池混合时间序列可能夸大趋势期效果 | P1 |
| recommendation | 已完成自选股范围推荐，但尚未接入全市场股票池和周期专用预测模型 | 当前最多只能从 4 只本地股票中筛选，长线结论仍以规则评分为主 | P1 |
| contracts | 已有标准契约快照，但尚未反向约束各模块输入输出 | 后续需要让 Agent、Web、实盘网关优先消费契约对象，减少字段分散 | P1 |
| web | 统一控制台首版已完成，但监控状态和单股深度钻取仍不足 | 使用者需要从单入口看完整证据链和任务健康状态 | P2 |
| monitoring | 本地产物监控和运行时任务日志已完成首版，仍缺少外部告警通道 | 长期运行还需要通知去重、冷却、趋势化和长驻调度子任务独立记录 | P1 |
| attribution | 已完成每日归因首版 | 后续需要接入更细行业、公告全文、真实资金流和多日连续归因链 | P1 |
| registry | 已完成数据注册表首版 | 后续需要各任务写入更细的缓存命中、补录来源、字段级质量和数据源版本 | P0 |
| agent | 已有 Provider/审计首版，但真实外部 LLM Provider 仍未接入 | 需要接入真实 LLM API、提示词模板和异常诊断任务 | P2 |
| docs | 总需求文档较大，执行顺序容易分散 | 需要保留一个唯一下一步表，减少并行发散 | P0 |

## 5. 实盘接入准备清单

| 类别 | 必备项 | 说明 |
|------|--------|------|
| 账户 | 券商 API、只读权限、模拟账户 | 先同步资金、持仓、委托、成交，不直接下单 |
| 行情 | 实时行情源、分钟线、涨跌停、停牌状态 | 下单前必须判断是否可交易 |
| 风控 | 单票上限、组合上限、行业上限、最大回撤、日内亏损熔断 | 与回测/模拟交易共用规则 |
| 执行 | 订单状态机、撤单、部分成交、失败重试 | 所有订单必须可追踪 |
| 审计 | 推荐证据包、人工确认、下单原因、版本快照 | 每笔订单能关联 prediction/selector/decision/review |
| 安全 | 密钥管理、权限隔离、只读/交易权限分离 | 禁止把密钥写入代码或报告 |

## 6. Agent 接入方式

| 步骤 | 内容 | 约束 |
|------|------|------|
| 1 | 定义 Agent 输入 Schema | 只允许读取标准证据包，不直接调用交易接口 |
| 2 | 增加 provider 抽象 | 支持本地规则、OpenAI 或其他 LLM provider 切换 |
| 3 | 增加 prompt/version 管理 | 每次输出保存模型、提示词版本、输入摘要和时间 |
| 4 | 增加安全门禁 | Agent 只能解释、归纳、提出建议，不能绕过 risk 和人工确认 |
| 5 | 增加审计回放 | 任意 Agent 输出可用同一输入复现或对比 |

## 7. v2 验收口径

- Selector 能根据 review/replay 自动校准，并在报告中解释阈值变化。
- Impact 数据能支撑“利润增长、材料涨价、产业价格事件”等真实影响分析。
- 实盘辅助先完成只读和审计，不进入自动下单。
- 推荐股票必须能回溯到大盘、资金、公告/业绩、产业价格、因子、回测和后验复盘。
- 未完成项必须继续进入本文“唯一执行顺序”，不得随意跳步。
